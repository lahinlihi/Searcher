## Selenium이란?

**Selenium** = 브라우저 자동화 도구

### 일반 크롤링 vs Selenium
```
일반 크롤링:
웹사이트 접속 → HTML 다운로드 → 파싱
(빠르지만 JavaScript로 로드되는 콘텐츠는 못 가져옴)

Selenium 크롤링:
웹사이트 접속 → Chrome 실행 → JavaScript 실행 → HTML 파싱
(느리지만 동적 콘텐츠도 가져옴)
```

### Selenium 사용 사이트 (4개)
- fanfandaero
- motie
- msit
- nia
- seoul-city (기존)
- iris (기존)

**영향**: 이 사이트들은 크롤링이 조금 느릴 수 있습니다 (1-3초 정도)

## 주의사항

### Selenium 사이트
- Chrome 드라이버가 필요합니다 (이미 설치됨)
- 크롤링 시 실제 브라우저가 백그라운드에서 실행됨
- 다른 사이트보다 느릴 수 있음

### SSL 인증서 문제
- koica, kosmes는 `verify_ssl: false` 설정됨
- 보안상 문제가 있을 수 있으니 주의

### 크롤링 실패 가능성
- 사이트 구조가 변경되면 셀렉터 수정 필요
- 사이트가 다운되거나 접근이 차단될 수 있음
- 0건 크롤링 시 셀렉터 재확인 필요

## 트러블슈팅

### 크롤링 결과가 0건인 경우
1. 웹 브라우저로 해당 사이트 직접 방문
2. F12 개발자 도구로 HTML 구조 확인
3. 셀렉터가 맞는지 확인
4. `data/settings.json`에서 셀렉터 수정

### Selenium 사이트 오류
1. Chrome이 설치되어 있는지 확인
2. ChromeDriver 버전 확인
3. 로그 파일 확인: `logs/` 폴더

### SSL 오류
- 해당 사이트를 비활성화하거나
- `verify_ssl: false` 설정 확인
