# '설정만' 사이트 분석 보고서

## 개요
현재 시스템에는 총 22개 사이트가 설정되어 있으며, 이 중 6개 사이트가 '설정만'으로 표시되어 있습니다.

'설정만'의 의미: 설정은 되어 있지만 UI에서 "크롤러 구현됨" 배지를 받지 못한 사이트들입니다.

---

## '설정만' 사이트 목록 및 현황

### 1. fanfandaero (한국중소벤처기업유통원)
**상태:** `enabled: false` (수동으로 비활성화됨)
**특징:**
- Selenium 사용 필요 (`use_selenium: true`)
- 셀렉터 설정: 완료
  - item: `table tbody tr`
  - title: `td a`

**조치 가능 여부:** ✅ 가능
- GenericCrawler로 크롤링 가능
- `enabled: true`로 변경하고 테스트 필요
- Selenium 동작 여부 확인 필요

**조치 방법:**
```json
"fanfandaero": {
    "enabled": true,  // false → true로 변경
    ...
}
```

---

### 2. koica (KOICA)
**상태:** `enabled: false` (수동으로 비활성화됨)
**특징:**
- SSL 검증 비활성화 (`verify_ssl: false`)
- 셀렉터 설정: 불완전
  - item: `tbody tr`
  - title: `td a`
  - link 셀렉터 누락

**조치 가능 여부:** ⚠️ 주의 필요
- GenericCrawler로 크롤링 가능
- link 셀렉터 추가 필요
- SSL 인증서 문제로 비활성화된 것으로 추정

**조치 방법:**
```json
"koica": {
    "enabled": true,
    "selectors": {
        "item": "tbody tr",
        "title": "td a",
        "link": "td a"  // 추가 필요
    },
    "verify_ssl": false
}
```

## 전체 요약

### 비활성화 이유 추정
1. **셀렉터 불완전**: 6개 사이트 모두 `link` 셀렉터가 누락됨
2. **SSL 인증서 문제**: koica, kosmes는 SSL 검증 비활성화 상태
3. **Selenium 의존성**: 4개 사이트(fanfandaero, motie, msit, nia)가 Selenium 필요

### 조치 가능 여부
| 사이트 | 즉시 활성화 가능 | 셀렉터 수정 필요 | 테스트 필요 |
|--------|----------------|----------------|------------|
| fanfandaero | ✅ | ❌ | ✅ |
| koica | ✅ | ✅ | ✅ |
| kosmes | ✅ | ✅ | ✅ |
| motie | ✅ | ✅ | ✅ |
| msit | ✅ | ✅ | ✅ |
| nia | ✅ | ✅ | ✅ |

## 활성화 권장 순서

### 1단계: fanfandaero (가장 간단)
- 셀렉터가 완전함
- `enabled: true`로 변경 후 바로 테스트 가능

### 2단계: koica, kosmes
- link 셀렉터 추가
- SSL 문제는 이미 `verify_ssl: false`로 처리됨

### 3단계: motie, msit, nia
- link 셀렉터 추가
- Selenium 동작 확인 필요 (시간이 더 걸림)

## 다음 단계 제안

1. **테스트 크롤링 실행**: 각 사이트를 개별적으로 테스트
2. **셀렉터 검증**: 실제 사이트 HTML 구조와 비교
3. **점진적 활성화**: 한 번에 하나씩 활성화하여 문제 확인
4. **implementedCrawlers 리스트 업데이트**: 작동 확인된 사이트를 리스트에 추가

## 기술적 배경

### '설정만' 표시 로직
- 파일: `static/js/settings.js`
- 조건: `implementedCrawlers` 배열에 포함되지 않은 사이트

### 크롤러 동작 원리
- 모든 '설정만' 사이트도 `GenericCrawler`를 통해 크롤링 가능
- `enabled: false`인 경우에만 실제 크롤링이 실행되지 않음
- 즉, UI 표시와 실제 기능은 별개

### 활성화 후 변경 사항
- 자동 크롤링 스케줄(09:00, 17:00)에 포함됨
- 수동 크롤링 버튼으로도 실행 가능
- 크롤링 결과가 데이터베이스에 저장됨
