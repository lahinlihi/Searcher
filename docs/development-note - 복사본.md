## 1. 프로젝트 개요

기존에 수작업으로 진행하던 정부 입찰공고 모니터링을 자동화하고, AI 기반 적합도 분석을 통해 관심 있는 공고를 신속히 발굴하기 위해 개발된 웹 시스템이다.

**핵심 목적**
- 나라장터(G2B) API, 사전규격, 다수의 정부·공공기관 사이트에서 공고를 자동 수집
- 관심 키워드 기반 적합도 점수를 계산하여 우선순위 제시
- 제안요청서(HWP/PDF) 내용을 Gemini AI로 분석하여 의사결정 지원
- 다수 사용자가 공유 접근하는 웹 서비스 형태로 운영

| **새 탭 열기** | 공고 링크(`/tender/:id`) 전체를 `target="_blank"`로 변경 (대시보드, 검색, 북마크, 검토공고) |
| **사전규격 → 본공고 감지 누락** | `deduplication.py`의 `existing_titles`에 사전규격 출처를 제외하도록 수정 → 사전규격과 동일 제목의 본공고가 중복 처리되어 누락되던 문제 해결 |
| **행사대행 분류 오류** | `scoring.py`에 `행사 대행` 패턴을 `행사운영` 카테고리에 추가 (`컨설팅` 보다 앞에 처리) |
| **수요기관 제외 필터 NULL 버그** | `NOT (NULL ILIKE '%패턴%')` = NULL → 행 제외 문제 수정, `db.or_(is_(None), ~ilike(...))` 적용 |
| **RPM 타이머 동결 + 이중 오류창** | `detail.html`의 `_showError()` 함수 개선: 타이머 표시 초기화, 두 오류창 동시 표시 방지 |
| **계약기간 HWP 원문 표시** | `_renderResult()` 우선순위 수정: Gemini 요약 결과 우선, 80자 초과 raw 텍스트 보조 처리 |
| **계약기간 앞 특수문자** | `normalizeContract()`에 선행 콜론·줄임표 제거 정규식 추가 |
| **Groq 입력 크기 최적화** | `max_chars=5000`, `max_tokens=2048`로 제한 |

- 지원 파일 형식: HWP / HWPX / PDF
- 분석 결과 DB 캐싱 (중복 API 호출 방지)

## 4. 기술 아키텍처

```
[외부 접속] https://ht-search.com
      │
      ▼
[Cloudflare Tunnel] ─── X-Forwarded-Proto: https 헤더 전달
      │
      ▼
[Waitress WSGI] localhost:5002
  trusted_proxy='*'
      │
      ▼
[Flask 3.0 + ProxyFix]
  app.py
  ├── routes/
  │   ├── auth.py       (로그인, OAuth, 회원가입)
  │   ├── admin.py      (관리자)
  │   ├── tenders.py    (공고 목록/검색 API)
  │   ├── analysis.py   (AI 분석 API)
  │   ├── bookmarks.py  (관심공고)
  │   ├── filters.py    (필터 설정)
  │   ├── settings.py   (사용자 설정)
  │   ├── data.py       (데이터 관리)
  │   └── pages.py      (정적 페이지)
  ├── crawlers/         (크롤러 모듈)
  ├── scoring.py        (점수 계산)
  ├── deduplication.py  (중복 제거)
  ├── document_analyzer.py (AI 분석)
  └── scheduler.py      (스케줄러)
      │
      ▼
[SQLite DB]  data/tenders.db
  - Tender (공고)
  - UserPreference (사용자 설정)
  - Bookmark (관심공고)
  - TenderAnalysis (AI 분석 캐시)
  - TenderNote (메모)
```

**배포 환경**
- 운영 서버: 개발자 PC (Windows)
- 공개 도메인: `https://ht-search.com`
- 터널: Cloudflare Tunnel (무료)
- 서버 프로세스: Waitress WSGI (8 threads)
- DB: SQLite (`data/tenders.db`)
- 자동 크롤링: 매일 오전 9시, 오후 5시 예약 실행
